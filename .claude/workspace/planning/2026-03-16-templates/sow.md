# SOW: テンプレートベースドキュメント生成

## Status

draft

## Overview

| Field      | Value                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------- |
| 目的       | chronicler にテンプレート内蔵 + 自動書き出し機能を追加し、人間向けドキュメント生成を完結させる |
| 対象       | src/template.rs (新規), src/templates/ (新規), src/config.rs, src/prompt.rs, src/main.rs       |
| アプローチ | `include_str!` でテンプレート内蔵 → 初回 check 時にファイル書き出し → パスをプロンプトに載せる |

## Background

前回の実装でchroniclerにinit/update/checkサブコマンドを追加し、LLM向けプロンプト出力を実現した。しかしプロンプトの内容が「ドキュメント作ってな」という汎用指示にとどまっており、生成されるドキュメントの構造・品質がLLM任せになっている。

既存の `/docs` スキルは4種類のテンプレート（architecture, api, domain, setup）で構造化された人間向けドキュメントを生成する仕組みを持つが、スキルはプラグインインストール時のみ利用可能。

chroniclerにテンプレートを内蔵し、check hookで自動的にテンプレートファイルを書き出すことで、ユーザーの明示的な操作なしにドキュメント生成の仕組みが整う。

## Acceptance Criteria

### AC-1: テンプレート自動書き出し

- [ ] check hook実行時にテンプレートディレクトリが存在しなければ、デフォルトテンプレート4ファイルを書き出す
- [ ] すでにテンプレートディレクトリが存在する場合は書き出ししない（ユーザー編集を尊重）
- [ ] 書き出し先は `{project_root}/{templates_dir}/`（デフォルト: `workspace/doc-templates`）

### AC-2: 設定による上書き

- [ ] `.claude/tools.json` の `chronicler.templates` でテンプレートディレクトリを指定できる
- [ ] 未指定時はデフォルト `workspace/doc-templates` を使用

### AC-3: テンプレートパス付きプロンプト

- [ ] initプロンプトにテンプレートファイルの絶対パス一覧を含める
- [ ] updateプロンプトに該当テンプレートの絶対パスを含める
- [ ] プロンプトは「テンプレートをReadして従って生成」という指示構造

### AC-4: init/check からの自動フロー

- [ ] check: テンプレートなし → 書き出し + initプロンプト（テンプレートパス付き）
- [ ] check: docsなし → initプロンプト（テンプレートパス付き）
- [ ] check: docs古い → updateプロンプト（テンプレートパス付き）
- [ ] init: テンプレートなし → 書き出し + プロンプト出力

## Risks

| リスク                             | 影響 | 軽減策                                                                  |
| ---------------------------------- | ---- | ----------------------------------------------------------------------- |
| テンプレート書き出しの権限エラー   | MED  | eprintln + graceful skip（書き出し失敗 → 内蔵パスなしでプロンプト出力） |
| バイナリサイズ増加                 | LOW  | 4 テンプレートで合計 ~10KB 程度                                         |
| テンプレート更新時のバージョン管理 | LOW  | 既存ディレクトリがあれば上書きしない方針                                |
