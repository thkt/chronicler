## Discoveries

- Stop hook + `"decision": "block"` は構造的に無限ループを引き起こす。Claudeがターンを終了できず、応答のたびにhookが再発火する。PreToolUse hookでは条件が解消可能なためblockは安全
- サブエージェント8体がchroniclerのstop hook (block mode) で全員ブロックされた。フックが自プロジェクトの開発中に自分自身をブロックする自己参照問題
- `glob::glob()` はシンボリックリンクを辿る。`scanner.rs` と `collector.rs` はガードしていたが `test_discovery.rs` は未対応だった
- formatterがPostToolUseでsource fileのmtimeを更新するため、gateのmtimeチェックに2秒toleranceが必要
- `find_refs_to_file` のbasename fallbackはadvisoryには許容だが、blocking gateではfalse positiveが致命的

## Design Changes

| Original Plan                               | Actual                                                           | Rationale                                                 |
| ------------------------------------------- | ---------------------------------------------------------------- | --------------------------------------------------------- |
| `mode: "block"` で stop hook を制御         | `mode` フィールド廃止、gate コマンドを新設                       | stop hook の block は無限ループ。PreToolUse gate で代替   |
| gate は staleness.rs の既存チェックを再利用 | gate 専用の inline staleness チェック（tolerance + exact match） | DA challenge: formatter race と basename 誤検出の回避     |
| 監査はサブエージェント8体で並列実行         | サブエージェント完了したが hook でブロック。Leader が直接統合    | chronicler stop hook が block mode でエージェントも止めた |

## Decisions

- gateはopt-in (`gate: false` default)。blockingは破壊的なUX変更なので明示的に有効化が必要
- gateはexact path matchのみ使用。basename fallbackはadvisory (edit) 専用
- gateのtoleranceは2秒。formatterがedit後ミリ秒以内に走ることを前提
- `run_edit` と `run_edit_test_docs` を `#[cfg(test)]` にしてバイナリのdead code warningを解消
- `lock::write_entry` と `lock::remove_entry` も `#[cfg(test)]` に（テスト専用関数）
- configの `load_both()` でtools.jsonの二重読み込みを解消
